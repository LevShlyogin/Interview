# База знаний «БРУСНИКА»

Корпоративная база знаний федеральной девелоперской компании «БРУСНИКА». Содержит регламенты, процессы, политики и справочные материалы для сотрудников офиса, стройплощадок, партнёров и управленческого звена.

## Назначение

База знаний предназначена для:

- онбординга новых сотрудников;
- ежедневной работы по регламентированным процессам;
- обучения RAG-систем и корпоративных чат-ботов;
- единого источника правды по ключевым бизнес-процессам компании.

## Метаданные документов

Каждый документ содержит YAML frontmatter с полями:

- `title`, `description`, `domain`, `audience`, `access`
- `owner`, `last_reviewed`, `status`, `tags`, `related`, `version`

Уровни доступа (`access`):

| Значение | Описание |
|----------|----------|
| `public` | Доступно всем, в том числе внешним пользователям |
| `internal` | Только сотрудники компании |
| `confidential` | Ограниченный круг (финансы, ценообразование) |
| `restricted` | Строго по ролям (ТЭО, финмодели) |

## Ключевые особенности компании

- **5 регионов:** Москва, Санкт-Петербург, Екатеринбург, Тюмень, Новосибирск
- **15–20 проектов** одновременно, ~800 000 м² в строительстве
- **~2 500** офисных сотрудников, **~4 000** на стройплощадках
- **Собственная СГП** — компания не привлекает внешних генподрядчиков

## Как пользоваться

1. Найдите нужный раздел по домену (`domain` в frontmatter).
2. Проверьте уровень доступа (`access`) перед передачей документа внешним пользователям.
3. Обратите внимание на `status`: документы со статусом `needs-review` или `archived` могут содержать устаревшую информацию.
4. Используйте `related` и раздел «Связанные документы» для навигации между связанными процессами.
