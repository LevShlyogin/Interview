# Этапы и ворота проекта (Stage-Gate)

Документ описывает систему контрольных точек (stage-gate) для управления портфелем девелоперских проектов «БРУСНИКА». Каждая «ворота» — формализованная точка принятия решения: продолжить, доработать или остановить проект. Обязателен для руководителей девелопмента, финансового блока и Инвестиционного комитета.

## Модель Stage-Gate

Проект проходит **5 ворот** (G0–G4) между **6 этапами**:

| Этап | Ворота | Ключевое решение |
|------|--------|------------------|
| 0. Лид / предоценка | G0 | Переход к due diligence |
| 1. Due diligence | G1 | Запуск ТЭО |
| 2. ТЭО и финмодель | G2 | Инвестиция (покупка земли) |
| 3. Проектирование и РНС | G3 | Старт строительства |
| 4. Строительство | G4 | Ввод и передача |
| 5. Продажи и эксплуатация | — | Закрытие проекта |

## Критерии прохождения ворот

### G0 → G1: Предоценка пройдена

- Карточка участка заполнена, ГПЗУ запрошен
- Нет критических юридических рисков (предварительная проверка)
- Региональный директор по девелопменту — **Go**

### G1 → G2: Due diligence завершён

- Юридический и технический отчёты DD без блокирующих замечаний
- Предварительная схема застройки согласована
- Бюджет на DD не превышен

### G2 → G3: Инвестиционное решение

- ТЭО с IRR ≥ 18%, NPV > 0
- Одобрение **Инвестиционного комитета** (протокол)
- Договорённости с банком по проектному финансированию и эскроу
- Земельный участок приобретён или зафиксирован опцион

### G3 → G4: Готовность к строительству

- РНС получен
- Бюджет B0 утверждён финансовым директором
- ПОС и ППР согласованы
- Начальник участка назначен, команда укомплектована

### G4: Ввод объекта

- ЗОС получен
- Пусконаладка завершена (см. [Пусконаладка](../construction/commissioning-process.md))
- Готовность к передаче клиентам и УК подтверждена

## Практический элемент: состав пакета для Инвестиционного комитета (G2)

Пакет подаётся за **5 рабочих дней** до заседания ИК:

1. ТЭО (финмодель, сценарный анализ) — доступ `restricted`
2. Отчёт юридического DD
3. Техническое заключение (геология, сети, ТУ)
4. Эскизный проект / ТЭП
5. Карта рисков с мерами митигации
6. Рекомендация регионального директора

**Формат решения ИК:** одобрить / одобрить с условиями / отклонить / отложить.

Срок рассмотрения: **1 заседание** (еженедельно по средам, центральный офис).
