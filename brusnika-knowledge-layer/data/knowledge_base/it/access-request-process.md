# Заявка на доступы к IT-системам

Регламент определяет порядок запроса, согласования и предоставления доступов к корпоративным информационным системам «БРУСНИКА». Для всех сотрудников, руководителей и HR при онбординге.

## Принципы

- **Минимальные привилегии** — доступ только к системам, необходимым для должности
- **Роль-based access** — типовые роли по грейду и подразделению
- **Согласование руководителя** — обязательно для всех заявок
- **Сроки** — стандартная заявка исполняется до **дня выхода** сотрудника

## Каналы подачи заявки

1. **Service Desk** (helpdesk.brusnika.ru) — основной канал
2. **HR-портал** — при онбординге (автоматическая пакетная заявка)
3. **Email** helpdesk@brusnika.ru — только при недоступности портала

## Типы заявок

| Тип | Инициатор | Согласующие | SLA |
|-----|-----------|-------------|-----|
| Онбординг (пакет) | HR | Руководитель (авто) | До дня 1 |
| Дополнительная система | Сотрудник | Руководитель + владелец системы | 2 р.д. |
| Повышение прав | Сотрудник | Руководитель + ИБ (при confidential) | 3 р.д. |
| Блокировка (увольнение) | HR | Автоматически | В день увольнения |
| Временный доступ | Сотрудник | Руководитель | 1 р.д. (макс. 30 дней) |

## Стандартные пакеты доступов

### Офис (все)

AD, почта M365, Teams, HR-портал, корпоративный портал, LMS

### По подразделениям

| Подразделение | Дополнительные системы |
|---------------|------------------------|
| Финансы | 1С (финансовый контур) |
| Коммерция | CRM «Продажи» |
| Девелопмент | CRM «Земля», Legal-portal (чтение) |
| СГП (офис) | Стройконтроль (чтение) |
| СГП (площадка) | СтройНЗ, ДефектСГП, СкладСГП |
| Юристы | Legal-portal, CRM (чтение) |

Доступ к **confidential** и **restricted** системам (прайс, ТЭО) — отдельная заявка с согласованием ИБ и владельца данных.

## Практический элемент: шаблон заявки

```
Тема: Доступ — [ФИО] — [система]
Подразделение: ...
Руководитель: ...
Система(ы): ...
Роль: [из справочника ролей]
Обоснование: ...
Срок (для временного): ...
```
