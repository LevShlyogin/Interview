# Политика информационной безопасности

Документ устанавливает требования по защите корпоративных данных, классификацию информации и правила работы сотрудников «БРУСНИКА» с информационными активами. Обязателен для всех сотрудников. Нарушение — дисциплинарная и материальная ответственность.

## Классификация данных

| Уровень | Описание | Примеры | Правила |
|---------|----------|---------|---------|
| Public | Публичная информация | Сайт, реклама | Без ограничений |
| Internal | Внутренняя | Регламенты, оргструктура | Только сотрудники |
| Confidential | Конфиденциальная | [Ценовая политика](../commercial/pricing-policy.md), вилки ЗП | NDA, запрет пересылки вне |
| Restricted | Строго ограниченная | [ТЭО](../development/feasibility-study-template.md), финмодели | По ролям, watermark, аудит |

## Обязательные правила

### Учётные записи и пароли

- Пароль: мин. 12 символов, смена каждые 90 дней
- Запрещена передача учётных данных третьим лицам
- MFA обязателен для VPN, почты, 1С, confidential-систем

### Рабочие устройства

- Ноутбуки офиса — только корпоративные (MDM)
- Запрещена установка нелицензионного ПО
- Шифрование диска (BitLocker) — обязательно
- На стройплощадках — корпоративные планшеты для «СтройНЗ»

### Email и мессенджеры

- Не открывать вложения от неизвестных отправителей
- Фишинг — сообщать на security@brusnika.ru
- Confidential данные — не отправлять по личной почте
- Клиентские ПДн — только через защищённые каналы CRM

### Физическая безопасность

- Не оставлять рабочее место без блокировки (Win+L)
- Confidential документы — не на принтере общего доступа
- USB-накопители — запрещены (кроме выданных ИБ)

## Инциденты ИБ

При подозрении на утечку, вирус, компрометацию пароля:

1. Немедленно сообщить security@brusnika.ru
2. Не выключать устройство (если ransomware — отключить от сети)
3. Сменить пароль (если применимо)
4. ИБ проводит расследование в течение **24 часов**

## Практический элемент: что нельзя делать

- Пересылать ТЭО и прайс-листы на личную почту
- Хранить ПДн клиентов в Excel на личном диске
- Обсуждать confidential информацию в личных мессенджерах
- Подключать личные устройства к корпоративной сети без MDM
- Передавать доступы коллегам «на время отпуска»
